# **DeskLegends Design System v1.0**

**Purpose**

This document defines the visual and interaction language of DeskLegends. Every interface, digital product, marketing asset, and customer touchpoint should follow these standards to maintain a consistent premium experience.



#### **1. Design Philosophy**

**Core Principle**

*The interface should never compete with the collectible.*

DeskLegends is an art gallery.

The artwork is always the hero.

The interface exists only to guide attention.



**Experience Goals**

Every visitor should feel:

1. Curiosity
2. Emotion
3. Craftsmanship
4. Trust
5. Aspiration
6. Ownership



**Design Keywords**

* Premium
* Museum
* Minimal
* Cinematic
* Elegant
* Timeless
* Confident
* Calm
* Immersive

Gold should never dominate.

It should highlight.

5–10% of the screen.

Motion is storytelling.

Not decoration.

**Easing**

#### **20. Code Standards**



HTML:

* Semantic elements
* Landmark roles
* BEM naming
* Logical heading hierarchy



CSS:

* Variables first
* Component-based
* Mobile-first
* No inline styles



JavaScript:

* Modular functions
* No global variables where avoidable
* Progressive enhancement
* Respect reduced-motion preferences



#### **21. Performance Budget**

* Initial page weight: < 2.5 MB
* Largest Contentful Paint (LCP): < 2.5 s
* First Contentful Paint (FCP): < 1.8 s
* Accessibility score: ≥ 95
* SEO score: ≥ 95
* Best Practices: ≥ 95



These targets help ensure the premium experience isn't compromised by slow loading.
